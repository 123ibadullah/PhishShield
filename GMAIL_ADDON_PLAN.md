# PhishShield AI: Enterprise Gmail Integration Plan

To transform PhishShield from a "copy-paste dashboard" into a **highly profitable, automated cybersecurity startup**, you must integrate directly into where users receive their emails: **Google Workspace (Gmail)**.

This plan details exactly how to pivot the current `api-server` architecture to support automated Gmail scanning via Google Apps Script and Gmail Add-ons.

---

## 1. The Architecture (How it will work)

1. **The Trigger:** A user opens an email in Gmail on Web or Mobile.
2. **The App Script:** The PhishShield Gmail Add-on (written in Google Apps Script/JavaScript) automatically hooks into the open email and reads the `from`, `subject`, and `body` (in text format).
3. **The API Call:** The Apps Script securely sends an HTTP `POST` request with the email payload to your `api-server` endpoint (`https://api.yourdomain.com/analyze`).
4. **The Machine Learning:** Your Node.js backend processes the email using the TF-IDF / Logistic Regression engine we built.
5. **The UI Injection:** The Add-on receives the JSON response (`riskScore`, `classification`) and injects a Native Google "Card" UI at the top of the email (e.g., a Big Red Banner saying **"⚠️ Phishing Detected"**).

## 2. The Implementation Scope

### Step A: Generate the Google Apps Script
You will create a new Google Cloud Project and write a basic `appsscript.json` manifest that declares permissions to read the user's email.

**appsscript.json (Required Permissions)**
```json
{
  "oauthScopes": [
    "https://www.googleapis.com/auth/gmail.addons.execute",
    "https://www.googleapis.com/auth/gmail.addons.current.message.readonly"
  ],
  "gmail": {
    "contextualTriggers": [{
      "unconditional": {},
      "onTriggerFunction": "buildAddOn"
    }]
  }
}
```

### Step B: The Google Apps Script (Code.gs)
This is the code that will live inside Google's servers. It pulls the currently open email and sends it to your Node.js backend.

```javascript
// Google Apps Script executed directly within Gmail
function buildAddOn(e) {
  var messageId = e.gmail.messageId;
  var accessToken = e.gmail.accessToken;
  
  GmailApp.setCurrentMessageAccessToken(accessToken);
  var message = GmailApp.getMessageById(messageId);
  var bodyDecoded = message.getPlainBody();

  // Call the PhishShield API we built exactly as we did in the Chrome Extension
  var apiUrl = "https://phishshield.yourdomain.com/api/analyze";
  var options = {
    method: "post",
    contentType: "application/json",
    payload: JSON.stringify({ emailText: bodyDecoded }),
    muteHttpExceptions: true
  };

  var response = UrlFetchApp.fetch(apiUrl, options);
  var apiData = JSON.parse(response.getContentText());

  return createCardUI(apiData);
}

function createCardUI(apiData) {
  var header = CardService.newCardHeader();
  var section = CardService.newCardSection();

  if (apiData.classification === "phishing") {
    header.setTitle("⚠️ HIGH RISK: Phishing Detected");
    header.setSubtitle("Risk Score: " + apiData.riskScore);
  } else if (apiData.classification === "suspicious") {
    header.setTitle("🚧 Suspicious Email");
    header.setSubtitle("Risk Score: " + apiData.riskScore);
  } else {
    header.setTitle("✅ Safe Email");
  }

  // List the reasons generated by your backend
  if (apiData.reasons && apiData.reasons.length > 0) {
    for (var i = 0; i < apiData.reasons.length; i++) {
      var textWidget = CardService.newTextParagraph().setText("• " + apiData.reasons[i].description);
      section.addWidget(textWidget);
    }
  }

  var card = CardService.newCardBuilder()
    .setHeader(header)
    .addSection(section)
    .build();

  return card;
}
```

## 3. What do you need to do to launch this?

1. **Deploy to a real Cloud Provider:** Your API backend currently lives on your local Windows PC at `localhost:5000`. Google Apps Script cannot talk to your local laptop. You must deploy your `api-server` Docker container to Render, Railway, or AWS.
2. **Add an API Key Authorization:** Right now, your `/analyze` route is completely public. You must add an Express middleware that checks for `Authorization: Bearer <API_KEY>` so only your Gmail Add-on is allowed to use your machine learning engine (so you don't pay for processing other people's data).
3. **Register on the Google Workspace Marketplace:** Once you publish the Add-on to the Google Marketplace, any startup or individual can install it via a single click. Every time they open an email, your Node.js server receives a ping, updates your SQLite database with the new threat intelligence, and securely protects their inbox.

This bridges the gap between a "hackathon project dashboard" and an **Enterprise B2B Product**.
